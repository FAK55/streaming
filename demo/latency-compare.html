Latency Comparison: WebRTC vs HLS

Same stream, same server, different protocols. Watch the delay difference.

WebRTC (WHEP) < 1 second
Protocol: WHEP (WebRTC) Port: 8889
HLS ~ 6 seconds
Protocol: HLS (.m3u8) Port: 8888

Why the difference?

HLS breaks video into small file segments (2-6s each). The player must download a segment before playing. WebRTC streams frame-by-frame over UDP, like a video call.

When to use HLS

Large audiences, broad device support, passive viewing. Works everywhere. Great when 6-second delay is acceptable (watching a tournament, VOD-style).

When to use WebRTC

Interactive features: live polls, predictions, chat-driven game mechanics, audience participation. Sub-second latency makes real-time interaction possible.

Setup: Start MediaMTX (docker compose up), then stream from OBS to rtmp://localhost:1935/live. Both players above will show the same stream with different latency.
